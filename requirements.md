# MediGuide AI — 需求规格说明书

## 1. 项目概述

**MediGuide AI** 是一套基于 Spring AI 的多智能体医疗健康系统，面向医疗机构和健康平台提供以下 AI 能力：
- 症状初步评估与分诊
- 药品信息查询与相互作用检查
- 健康数据摘要与趋势分析
- 慢性病长期管理指导

## 2. 功能需求

### 2.1 症状分诊 (Symptom Triage)
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F1 | 用户输入症状、持续时间、年龄等，进行紧急程度分级 | P0 |
| F2 | 红色警报症状自动识别（胸痛、呼吸困难、大出血等） | P0 |
| F3 | 提供分级对应的行动建议（就医/居家观察） | P0 |
| F4 | 特殊人群（儿童、老人、孕妇）自动升一级 | P1 |

### 2.2 用药指导 (Medication Guide)
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F5 | 药品基本信息查询（用途、剂量、副作用） | P0 |
| F6 | 药物相互作用检查（两药/多药联用） | P0 |
| F7 | 常见用药误区提醒 | P1 |
| F8 | 药品说明书结构化解析 | P2 |

### 2.3 健康摘要 (Health Summary)
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F9 | 用户体征数据录入/导入 | P0 |
| F10 | 体征趋势分析（血压/血糖/体重） | P0 |
| F11 | 异常指标自动标注和建议 | P1 |
| F12 | 生成可打印的健康报告 | P2 |

### 2.4 慢病管理 (Chronic Care)
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F13 | 疾病管理指南（糖尿病/高血压/COPD/哮喘） | P0 |
| F14 | 每周自我管理清单生成 | P0 |
| F15 | 用药提醒与复诊提醒 | P1 |
| F16 | 生活方式建议（饮食/运动/睡眠） | P1 |

## 4. 数据流

```
User Input → MedicalAdvisorService
  ├→ RAG Enrichment (向量检索相关医疗知识)
  ├→ Category Router
  │   ├→ SymptomTriageAgent → EmergencyTriageTool → Response
  │   ├→ MedicationGuideAgent → DrugInteractionTool → Response
  │   ├→ HealthSummaryAgent → HealthDataTool → Response
  │   └→ ChronicCareAgent → ChronicCareTool → Response
  └→ Unified Response (含分诊等级 + 建议 + 免责声明)
```

## 5. 非功能需求

| 类别 | 要求 |
|------|------|
| 响应时间 | P0接口 < 5s，P1接口 < 10s |
| 并发 | 支持100+并发查询 |
| 可用性 | 99.9%（核心分诊服务）|
| 安全 | 所有医疗数据加密传输和存储 |
| 合规 | 输出必须含免责声明，不含诊断结论 |
| 可审计 | 所有AI回复留存日志 |

## 6. 依赖清单

- **Spring AI 1.0.4** — AI 模型集成框架
- **Spring Boot 3.4** — 应用框架
- **Ollama** — 本地 LLM 推理
- **PGVector** — 向量数据库
- **PostgreSQL 17** — 关系数据库
- **Redis 7** — 会话缓存
- **JDK 21+** — 运行时
- **Maven 3.9+** — 构建工具
- **Docker Compose** — 基础设施编排

## 7. 安全与合规

- HIPAA 合规（美国）/ 个保法（中国）
- 数据最小化原则：不存储不必要的健康数据
- 所有 API 需认证（企业版）
- AI 回复必须包含免责声明
- 禁止提供明确诊断
- 紧急情况引导用户就医而非自行处理
